# Документация
0. Спрашивай в конце работы - нужно ли редактировать документацию
1. Веди документацию в MD файлах (например BUSINESS.MD, STORAGE.MD и т.д.) для каждого модуля (src/lib/[api,business,components и т.д.]).
2. Обновляй документацию, если что-то изменилось в ходе работы.
3. Храни файлы документации в папке модуля, рядом сопутствующим кодом.
4. Делай документацию локаничной.
